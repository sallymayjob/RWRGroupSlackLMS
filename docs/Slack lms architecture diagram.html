The 4 Workflow Patterns (n8n implementations)

WF-A

Course Launcher

Agent 08 — /enroll
  • Create learner row in PostgreSQL
  • Set month=1, status=active
  • Send welcome + "Start Month 1"
  • Write enrolled_at timestamp
■ STOP — no lessons inside
WF-B

Lesson Delivery

Agent 03 — /learn
  • Parse lesson_id from command
  • Fetch content from Notion API
  • Format as ULC Slack message
  • Post to learner DM
↻ Reused for every lesson
WF-C

Progression Checker

Agent 02 — /submit
  • Redis dedup lock (30s)
  • Write w{n}_completed = true to PG
  • Check: all 4 weeks complete?
  • If YES → month++, celebrate
🧠 Replaces ALL delays
WF-D

Utility Commands

Supervisor Router
  • /progress — visual bar
  • /help — command reference
  • /unenroll — set withdrawn
  • /backup — trigger Sheet export
